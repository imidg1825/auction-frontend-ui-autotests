# Сценарии UI-автотестов

Документ описывает логические области покрытия автотестами интерфейса аукциона/объявлений: что проверяется, какие пользовательские сценарии им соответствуют и при каких расхождениях имеет смысл заводить баг.

---

## Главная страница

**Что покрывается:** загрузка главной, ключевые элементы шапки (поиск, категории, создание объявления), отображение ленты карточек, переход в карточку товара/аукциона с главной.

**Пользовательские сценарии:** открытие сайта; гость видит вход; пользователь видит привычную шапку и выдачу; переход к объявлению с главной.

**Основание для бага:** главная не открывается или открывается с ошибкой; отсутствуют ожидаемые элементы после загрузки; клик по карточке не ведёт на страницу объявления; неверное поведение для гостя и авторизованного пользователя.

---

## Авторизация и регистрация

**Что покрывается:** открытие модального окна входа, ввод e-mail, валидация и шаги до экрана кода/регистрации, поля и согласия на форме регистрации (в рамках доступного без реального OTP).

**Пользовательские сценарии:** вход по e-mail; переход к вводу кода; начало регистрации и заполнение персональных данных в UI.

**Основание для бага:** модалка не открывается; кнопки/поля ведут себя иначе, чем задумано (недоступны при корректных данных); сломан текст или порядок шагов; регрессия доступности формы.

---

## Поиск и фильтры

**Что покрывается:** поле поиска в шапке, отправка запроса (кнопка и Enter), отображение выдачи и карточек, переход в объявление из поиска, категории с главной, переключатели типа выдачи (если есть на стенде), пустая выдача по заведомо «пустому» запросу.

**Пользовательские сценарии:** найти объявления по запросу; сузить выдачу категорией; переключить тип списка; увидеть корректное пустое состояние.

**Основание для бага:** поиск не запускается или не обновляет контент; неверный URL/состояние SPA при поиске; категории не открываются или не меняют выдачу; отсутствует понятное сообщение при отсутствии результатов.

---

## Карточка товара

**Что покрывается:** отображение страницы объявления (фиксированная цена): ключевые блоки, кнопки действий, связанные переходы, базовая консистентность UI.

**Пользовательские сценарии:** просмотр объявления; взаимодействие с доступными без бэкенд-данных элементами.

**Основание для бага:** страница не загружается; критичные блоки или CTA отсутствуют; неверные данные в шапке/основном контенте; сломанные ссылки или состояния кнопок.

---

## Создание объявления

**Что покрывается:** вход в сценарий с главной или по `/createAd`, выбор категории, поля формы (название, описание, фото, цена/аукцион, местоположение), по возможности полный сценарий публикации.

**Пользовательские сценарии:** создать объявление с нуля; выбрать тип размещения; заполнить обязательные поля и отправить форму.

**Основание для бага:** сценарий ведёт на логин вместо формы для авторизованного пользователя; шаги категории или формы недоступны; валидация или кнопка публикации работают неверно; после отправки нет ожидаемого результата на стенде.

---

## Профиль

**Что покрывается:** раздел личного кабинета/профиля: доступность из UI, базовая структура страницы и навигация, соответствие ожидаемым заголовкам и зонам контента.

**Пользовательские сценарии:** авторизованный пользователь открывает профиль и ориентируется в разделе.

**Основание для бага:** профиль не открывается; пустой или сломанный layout; неверные заголовки/роли элементов; регрессия после релиза.

---

## Избранное

**Что покрывается:** переход в избранное, заголовок раздела, отображение карточек или пустого состояния, вход из меню профиля (если предусмотрено тестами).

**Пользовательские сценарии:** просмотр сохранённых объявлений; понимание, что список пуст.

**Основание для бага:** раздел недоступен; неверный URL; нет карточек при наличии избранного или нет пустого состояния при отсутствии; сломана навигация из профиля.

---

## Сообщения

**Что покрывается:** раздел сообщений из меню профиля, список диалогов, открытие чата, поле ввода и отправка (в т.ч. только для диалогов с активным полем ввода), пустое состояние при отсутствии переписок.

**Пользовательские сценарии:** открыть переписки; написать сообщение в доступном диалоге.

**Основание для бага:** сообщения не открываются; поле ввода недоступно там, где чат должен быть активным; потеря списка диалогов; регрессия после изменений чата.

---

## Уведомления

**Что покрывается:** доступ к разделу/колокольчику уведомлений, отображение списка или пустого состояния, базовая работоспособность UI.

**Пользовательские сценарии:** пользователь проверяет уведомления.

**Основание для бага:** уведомления не открываются; пустой экран без объяснения; ошибки при прокрутке или клике по элементам списка.

---

## Мои объявления

**Что покрывается:** раздел объявлений пользователя: табы/списки (например, на модерации, активные), наличие карточек или корректных состояний списка.

**Пользовательские сценарии:** просмотр своих объявлений и статусов.

**Основание для бага:** раздел не загружается; неверные вкладки или счётчики; объявления не отображаются при наличии данных на стенде.

---

## Мои ставки

**Что покрывается:** раздел ставок пользователя, заголовок и контент списка (ставки, пустое состояние).

**Пользовательские сценарии:** пользователь проверяет историю/активные ставки.

**Основание для бага:** раздел недоступен; неверная структура страницы; отсутствует список или пустое состояние там, где ожидается контент.

---

## Аукцион

**Что покрывается:** переход на страницу аукциона с главной (при наличии карточки), отображение ставки, таймера, валюты, кнопки ставки; модальное окно ввода суммы и активация подтверждения.

**Пользовательские сценарии:** участник просматривает аукцион и открывает диалог ставки.

**Основание для бага:** страница аукциона не открывается или не совпадает с ожидаемым типом объявления; отсутствуют ключевые блоки; модалка ставки не открывается или кнопки в неверном состоянии.

---

## Негативные сценарии

**Что покрывается:** обработка ошибочных или граничных действий (некорректный ввод, недоступные действия без данных, сообщения об ошибках в формах).

**Пользовательские сценарии:** пользователь совершает ошибку и должен получить понятную обратную связь от UI.

**Основание для бага:** нет валидации там, где она нужна; падение интерфейса без сообщения; неверный или отсутствующий текст ошибки.

---

## Настройки профиля

**Что покрывается:** экран настроек профиля: доступность полей, переключателей и сохранения (в рамках безопасных для автотеста действий).

**Пользовательские сценарии:** пользователь меняет настройки аккаунта/уведомлений и сохраняет их.

**Основание для бага:** настройки не открываются; элементы не реагируют на действия; сохранение не отражается в UI или ломает страницу.
